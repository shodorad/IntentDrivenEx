13 — Guidelines

Do & Don't

Rules to keep the design language consistent and professional across all components.

Do
  • Use compound shadows (tight contact + soft diffused) for card elevation
  • Use navy (#0D2137) for primary CTAs and user bubbles — it's the anchor
  • Use teal (#00B5AD) for interactive accents, data states, and AI branding
  • Keep card border-radius at 14px consistently across all tile types
  • Use 3-column grid for alerts so they're always symmetrical
  • Put severity in the icon background color, not a colored left border
  • Add 1px border (rgba(0,0,0,0.04)) to cards even with shadow — adds crispness
  • Use Inter font at 700–800 weight for data numbers and headlines
Don't
  • Don't use teal (#00B5AD) as a full card background — it overwhelms the hierarchy
  • Don't use flat single-layer box-shadow (e.g. 0 1px 5px) — it looks cheap
  • Don't use left border accents on alert cards — they look like error logs, not a consumer app
  • Don't mix inline styles and CSS modules for the same component
  • Don't use #9ca3af for any visible text — fails WCAG AA on white. Use #6b7280 minimum
  • Don't use Total Red (#CC0000) anywhere except the logo
  • Don't stack alerts vertically in a column — always use horizontal grid
  • Don't render the same signal in both SignalBanner and SignalRow

⚠️ Issues Found — Root Cause Summary

  • BUG Explore pills all break for Maria. "Show me all plans", "Latest phones", "Current deals", "My rewards" all trigger Maria's low-data opening instead of relevant content. The isPlanBrowse/isPhoneBrowse checks use literal substring matching that doesn't match the actual pill prompt strings.
  • BUG getMariaTurnResponse() routes by Turn 1 message, not current message. If Maria's opening triggers from "Quick Refill — $15" (Turn 1), ALL turn-2 pills (including "Why am I running out?", "Change my plan", "I'm fine for now") route through the refill path regardless of what the user actually clicked.
  • BUG Same issue with "Change my plan" as Turn 1. Any Turn 1 entry via "Change my plan" causes ALL turn-2 responses to show the Unlimited upgrade card, regardless of what pill the user tapped.
  • UX Post-diagnosis upgrade path has no continuation handlers. After showing upgrade options ("Start now — ~$7 today", "Switch on Apr 9 — free"), there is no specific handler. Falls through to generic refill fallback which gives the wrong copy.
  • UX "I'm fine for now" exits to a dead end. This pill from Maria's opening results in the generic refill suggestion on the next turn — ignoring the user's stated "I'm fine" intent.
  • UX "Cancel" pill has no handler. Clicking Cancel on the refill confirmation doesn't dismiss the flow or offer a clear exit — falls to the generic refill prompt on next turn.
  • UX Info alert CTA routes wrong. The "Tell Me More" CTA on the info card ("App opened 3 times today") sends the signal headline as a prompt, which isn't recognized and shows Maria's standard opening — creating confusion.

📋 Research Notes for Fix Planning:
The root cause of most issues is a single architectural pattern: getMariaTurnResponse() determines paths using the first user message (turn 1 intent), not the current message. This means the entry pill "contaminates" all downstream routing. The fix needs to either (1) route by current message at each turn, not first message, or (2) surface the correct response immediately on turn 1 instead of showing the generic Maria opening first. The Explore pill bugs are a simpler text-matching issue in the browse detection regexes.
